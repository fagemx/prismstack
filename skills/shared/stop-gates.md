# STOP Gates — 互動設計規範

## Placement Rules
1. One STOP gate per phase boundary (minimum)
2. Extra gates for: long phases (>10 min work), risky decisions, irreversible actions
3. Never run more than 5 skills or 5 minutes of work without stopping

## What to Present at Each Gate
Using AskUserQuestion format (see ask-format.md):
1. Summary of what was just completed
2. Key findings or decisions made
3. What's about to happen next
4. Options: continue / adjust / stop

## Combining with AskUserQuestion
STOP gate = mandatory AskUserQuestion. The gate does not auto-continue.
User must explicitly choose to proceed.

## Recovery After STOP
If user pauses and returns later:
- Re-read the current state (artifacts, progress file)
- Re-ground (remind user where we are)
- Offer to continue or restart the current phase
